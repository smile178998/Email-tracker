
# Email Tracker - 邮件/文件打开追踪工具

当对方打开你的邮件或文件时，自动记录并显示对方所在的地理位置（城市、省份、国家）。

## 功能特点

- **PNG 像素追踪**：对方打开邮件时自动触发（适用于允许加载图片的邮件客户端）
- **SVG 像素追踪**：使用 SVG 图片触发，部分客户端可能允许
- **链接点击追踪**：对方点击链接时触发，绕过图片屏蔽，获取真实 IP
- **地理位置显示**：自动获取IP地址对应的城市、省份、国家、时区
- **实时面板**：5秒自动刷新，实时查看追踪记录

## 安装步骤

1. 安装依赖：
```bash
npm install
```

2. 启动服务器：
```bash
npm start
```

3. 访问控制面板：
```
http://localhost:3000/dashboard.html
```

## 使用方法

### 1. 创建追踪器
在控制面板输入追踪器名称（例如："报价单邮件-客户A"），点击"新建追踪器"。

### 2. 复制追踪代码
展开追踪器，复制对应的HTML代码：

**PNG 像素追踪**：
```html
<img src="http://your-server:3000/pixel/abc123" width="1" height="1" style="display:none">
```

**SVG 像素追踪**：
```html
<img src="http://your-server:3000/svg/abc123" width="1" height="1" style="display:none">
```

**链接点击追踪**（推荐）：
- 输入目标URL（如：https://example.com/file.pdf）
- 点击"生成"按钮
- 复制生成的追踪链接

### 3. 嵌入到邮件或文件
- **邮件**：将图片像素代码粘贴到邮件HTML中
- **文件**：将追踪链接作为文件下载链接发送

### 4. 查看追踪记录
对方打开邮件或点击链接后，控制面板会自动显示：
- 打开时间
- IP地址
- 城市
- 省份
- 国家
- 时区
- User-Agent（设备信息）

## 各追踪方法的局限性

### PNG/SVG 像素追踪的局限性

**Gmail 用户**：
- Gmail 会代理所有图片请求，因此 IP 地址将显示为 Google 服务器的 IP，而非收件人的真实 IP
- 无法获取收件人的真实地理位置

**Outlook 桌面版**：
- 默认屏蔽外部图片，像素追踪无法触发
- 需要用户手动启用"下载图片"功能

**其他客户端**：
- 部分客户端可能屏蔽图片或使用代理
- 触发率受客户端隐私设置影响

### 链接点击追踪（唯一可靠方案）

**优势**：
- ✅ Gmail：用户点击链接时直接访问服务器，获取真实 IP
- ✅ Outlook 桌面版：绕过图片屏蔽，完全可靠
- ✅ 所有客户端：只要用户点击链接即可触发
- ✅ 获取真实地理位置

**局限性**：
- 需要用户主动点击链接（非自动触发）
- 需要在邮件中提供有吸引力的链接内容

## 推荐使用策略

1. **Gmail 用户**：使用链接点击追踪，因为像素追踪只能获取 Google IP
2. **Outlook 桌面版**：必须使用链接点击追踪，像素追踪无法工作
3. **其他客户端**：可组合使用像素追踪 + 链接点击追踪提高触发率
4. **重要追踪**：始终使用链接点击追踪作为主要方法

## 注意事项

- 需要公网服务器才能对外追踪（本地localhost无法被外部访问）
- 追踪数据保存在 `logs.json` 文件中
- 请遵守当地法律法规，合法使用
- Gmail 的图片代理是隐私保护机制，无法绕过

## 部署到公网

使用 ngrok 等内网穿透工具快速测试：
```bash
ngrok http 3000
```

或部署到云服务器（如阿里云、腾讯云、AWS等）。
